compMap — Server-Driven Layout — Type Interlinkings

Main Decoding Flow

JSON from server
    │
    │  { "type": "box", "data": { ... } }
    ▼
SearchComponentType.init(from decoder:)          SearchComponentType.swift
    │  reads "type" key → routes to BoxComponent.type ("box")
    │  decodes "data" → BoxComponentModel
    ▼
SearchComponentType.box(BoxComponent)            ← one of ~55 enum cases
    │
    │  .view  →  BoxComponentView
    │  .model →  BoxComponentModel
    ▼
BoxComponent                                     BoxComponent.swift
    │  static let type = "box"   (string key used in decoder routing)
    │  let model: BoxComponentModel
    │  var viewModel: BoxComponentViewModel
    ▼
BoxComponentModel  (@APIv3Model decoded)         BoxComponentModel.swift
    ├── components: [SearchComponentType]   ◄──── RECURSIVE: each child
    │                                             goes through the same
    │                                             SearchComponentType decoder
    ├── layout: SearchLayout?               ◄──── server drives this
    └── style:  SearchStyle?
                    │
                    └── padding: SearchPadding?
                            (top / left / bottom / right / horizontal / vertical)

SearchLayout  (decoded from "layout" field)      Layout.swift
    ├── type: LayoutType                   ◄──── decoded from "type" key
    │       .grid | .carousel | .list
    │       .stack | .standalone
    │
    └── attributes: LayoutAttributes?      ◄──── decoded from "data" subkey
            │  (protocol, type-erased)
            │
            ├── GridLayoutAttributes        GridLayoutAttributes.swift
            │       size:        Int        (columns, default 2)
            │       rowSpacing:  CGFloat
            │       itemSpacing: CGFloat
            │       alignment:   LayoutAxisAlignment
            │
            ├── CarouselLayoutAttributes    CarouselLayoutAttributes.swift
            ├── ListLayoutAttributes        ListLayoutAttributes.swift
            ├── StackLayoutAttributes        StackLayoutAttributes.swift
            └── StandaloneLayoutAttributes  StandaloneLayoutAttributes.swift

Recursive Nesting

SearchComponentType.box
  └── BoxComponentModel
        ├── layout: SearchLayout (grid)
        └── components:
              ├── SearchComponentType.box
              │     └── BoxComponentModel
              │           ├── layout: SearchLayout (carousel)
              │           └── components: [recentSearches, trendingSearches, ...]
              └── SearchComponentType.sectionTitle(...)

Two-Layer Split

Layer Type File Responsibility
Decode SearchComponentType SearchComponentType.swift Routes JSON "type" key → right model
Raw data BoxComponentModel BoxComponentModel.swift Holds decoded [SearchComponentType] + SearchLayout
Runtime BoxComponent BoxComponent.swift Wraps model, owns id, creates VM / View
Presentation BoxComponentViewModel BoxComponentViewModel.swift Defaults SearchLayout to .standalone if nil
Layout config SearchLayout Layout.swift Decodes layout "type" + "data" attributes
Layout impl XxxLayoutAttributes SearchLayout/Layouts/*/ Type-specific spacing / size values
